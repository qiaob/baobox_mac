# Baobox

[![CI](https://github.com/qiaob/baobox_mac/actions/workflows/ci.yml/badge.svg)](https://github.com/qiaob/baobox_mac/actions/workflows/ci.yml)

菜单栏常驻的 macOS 效率工具集：把截图、剪贴板管理、取色器等日常工具整合进一个原生 App，统一入口、统一快捷键、统一设置。

[English](README.md) · [简体中文](README.zh-CN.md)

## 概述

主流 macOS 效率工具大多各自独立成一个 App：截图一个、剪贴板管理一个、取色器又一个——各自下载、各自常驻后台、各自维护一套设置。Baobox 反其道而行：用 Swift 原生开发单一菜单栏 App，内部由多个相互独立的工具模块组成，对外统一呈现一套一致的交互。

- **原生实现，非 Electron** —— Swift 5.9、SwiftUI + AppKit 混合、零第三方依赖，常驻内存低于 ~50MB。
- **模块化架构** —— 应用框架不认识任何具体工具，每个工具实现统一的 `ToolModule` 协议，自行注册菜单项、快捷键与设置页。
- **数据本地优先** —— 剪贴板历史、截图、取色记录、窗口布局等全部数据仅保存在本机磁盘，不上传任何服务器。
- **系统要求 macOS 14 (Sonoma) 及以上**，依赖 ScreenCaptureKit 等现代系统框架。

## 功能列表

### 框架
- 菜单栏常驻，无 Dock 图标（`LSUIElement`）；菜单为纯工具列表：每个工具一行（图标 + 名称 + 主快捷键），悬停展开二级菜单承载操作、历史与设置入口。
- 全局快捷键中心（Carbon）：统一注册、冲突检测、自定义、持久化。
- 统一设置窗口：通用 / 快捷键 / 每个工具一个 Tab / 关于。
- 首次启动权限引导，授权状态实时刷新。
- 可选开机自启（`SMAppService`）。

### 截图（默认 ⌘⇧2）
- 单一快捷键，自动识别意图：悬停高亮窗口，单击即截取；按下拖拽（超过约 4pt 阈值）切换为区域截图，支持八向手柄微调、方向键逐像素移动（⇧ ×10）与像素放大镜；⏎ 全屏截图；Esc 取消。
- 多显示器支持，每屏独立 overlay。
- 基于 ScreenCaptureKit；结果复制到剪贴板，并可按设置保存到指定目录（文件名模板可配置）。
- 标注编辑器：截取后原地标注——矩形、椭圆、箭头、画笔、荧光笔、马赛克、文字、橡皮，支持撤销重做（⌘Z / ⇧⌘Z），三档粗细 + 七色色板。
- 贴图（Pin）：把截图钉在屏幕最上层，可拖动，滚轮缩放（0.2×–5×）、⌥+滚轮调透明度；支持直接从剪贴板贴图。
- 像素放大镜：悬停、框选或拖拽手柄时跟随光标，17×17 物理像素 ×8 放大，附带实时坐标与颜色 Hex 读数。
- 截图历史：每次截图自动存档（保留张数可配置，默认 20），菜单缩略图列表支持复制 / 贴到屏幕 / 另存 / 删除。
- 屏幕录制：复用截图的选区交互（框选区域 / 点选窗口 / 全屏），输出 MP4 或 GIF；可选录制系统声音与麦克风（双轨默认自动混为单轨）；录制中以红色边框圈出范围，悬浮控制条支持暂停 / 继续 / 停止。

### 剪贴板（默认 ⌘⇧V）
- 后台监听剪贴板，记录文本、富文本、图片、文件路径与链接；标记为隐私的内容（`org.nspasteboard.ConcealedType` / `TransientType`）不会入库。
- 浮层历史面板：即输即搜、按类型过滤、上下键选择、⏎ 粘贴、⌥⏎ 纯文本粘贴、⌘P 置顶。
- 选中条目后通过模拟 ⌘V 自动回填粘贴（需要辅助功能权限）；未授权时降级为仅复制。
- 历史条数上限可配置，超限自动淘汰；重启后保留。
- 支持按 App 设置忽略名单、过期自动清理（永久 / 1 / 7 / 30 / 90 天）、单条删除（⌘⌫ 或悬停删除按钮）；全局 ⌘⌥V 可直接以纯文本粘贴最近一条。

### 取色器（出厂不绑定快捷键）
- 基于系统原生 `NSColorSampler` 放大镜取色，无需任何权限。
- 取色结果按所选格式（Hex / RGB / SwiftUI `Color`）复制到剪贴板，并存入历史（上限 50 条）。
- 二级菜单展示最近 5 个颜色色块，点击即按当前格式复制。
- 设置页可切换输出格式、Hex 大小写、取色后是否自动复制。

### 防休眠（纯菜单操作）
- 基于 IOKit 电源管理断言（`IOPMAssertionCreateWithName`）阻止系统进入空闲休眠。
- 二级菜单可开启 15 分钟 / 1 小时 / 2 小时 / 无限期，到期自动关闭，菜单实时显示剩余时间。
- 可选「同时防止显示器休眠」；App 退出时自动释放断言。

### 窗口管理（出厂不绑定键位）
- 基于辅助功能权限移动 / 缩放前台窗口：半屏、四分屏、最大化（非全屏）、居中、跨显示器移动、恢复原始位置，共 13 条可自定义快捷键（出厂不绑定——常用的 ⌃⌥ 系列与 Rectangle 完全同键，请在设置 → 快捷键中自行设置）。
- **布局快照**：「保存当前布局…」记录所有常规、非最小化窗口的位置与尺寸，菜单一键恢复，标题优先匹配、顺序兜底，未运行的 App 自动跳过。
- **完整多显示器兼容**：目标屏取与窗口交集面积最大者；布局基准统一使用各屏的可见区域（自动避开菜单栏与 Dock）；跨屏移动按相对位置与尺寸等比映射并做边界裁剪，不同分辨率或排列之间不会变形溢出；快照记录每屏稳定的 UUID 与屏内相对位置，即便分辨率或显示器排列发生变化，恢复时依然准确。

### 二维码生成（默认 ⌃⇧Q）
- 唤起浮层自动带入当前剪贴板文字，内容修改后实时重新生成（纠错等级 M，含静区）。
- 支持复制为图片、保存 PNG，或直接钉在屏幕上。
- 纯本地实现（`CIQRCodeGenerator`），无需任何权限。

### Claude Code 助手（出厂不绑定快捷键）
- 为本地 Claude Code CLI 打造的菜单栏仪表盘，数据全部来自 `~/.claude` 下的本地文件——不调用任何 AI API、无需登录：实时会话状态（运行中 / 等待确认）、最近 5 个会话一键在终端续接、当前 5 小时额度窗口（token 用量、估算花费、重置倒计时）与今日估算花费。
- **中心窗口**（会话 / 用量 / 审计）：可搜索的会话历史，支持续接、复制续接命令、导出 Markdown 与删除；按天 / 项目 / 模型的用量报表，外加调用统计（Skill 与斜杠命令、MCP 服务器 › 工具、内置工具）；以及按日的文件改动审计，可在访达中定位被改文件。
- **后台增强**（按需开启）：Baobox hooks 在任务完成或 Claude 等待确认时发送系统通知，用量达 token 预算 80% 时提醒；危险命令卫士在 `rm -rf`、`git push --force`、`git reset --hard`、`DROP TABLE` 等命令执行前拦截，并把原因反馈给 Claude。
- **可视化配置**：权限模式 / 默认模型 / 会话保留天数选择器、权限预设勾选、隐私环境开关、Co-Authored-By、CLAUDE.md 管理、带实时预览的 statusline 生成器、MCP 服务器面板，以及磁盘清理与版本检查——所有写入都安全处理用户 JSON（保留未知字段，写前备份 `.baobox.bak`）。
- 费用为按公开定价的估算值，界面处处标注「估算」。

### Codex 助手（纯菜单操作）
- 为本地 Codex CLI 打造的助手，数据全部来自 `~/.codex` 下的本地文件——不调用任何 AI API、无需登录。菜单顶部为状态行（会话数 + 今日估算花费），下方列出最近的 Codex 会话供一键在终端续接（`codex resume <id>`），并有「浏览会话 / 用量…」入口打开带 Tab 的中心窗口。
- **用量与额度**：把 rollout JSONL 里的 Codex `token_count` 事件聚合为滚动 5 小时窗口与周窗口（默认近 168 小时滚动，或按固定星期/小时重置对齐），并统计今日累计。「用量报表…」菜单项带两行副标题（5 小时 + 本周：已用 ≈ 估算花费 · 距重置倒计时）。增量型（`last_token_usage`）与累计型（`total_token_usage`）两种记账都能正确处理，避免双计。
- **中心窗口**：**会话** Tab（搜索、续接、复制命令、删除）与**用量** Tab（5 小时 / 本周窗口卡片，可选预算进度条；按天 / 按项目 / 按模型三维报表；内置工具 / MCP 调用统计）。费用来自内置定价表，均标「估算」。
- **Codex 配置可视化**：`approval_policy`、`sandbox_mode`（danger 档红字警示）与默认 `model` 以选择器编辑，对 `config.toml` 做行级读写，保全注释与未知键；遇到无法安全编辑的值会自动识别并置灰对应控件。每次写入前备份 `config.toml.baobox.bak`。
- **完成通知**（按需开启）：把 `notify` 钩子写入 `config.toml`；Codex 回合结束时 Baobox 发送系统通知（含最后回复摘要），移除后 config.toml 无残留。
- **维护**：展示 `~/.codex/sessions` 磁盘占用与文件数，清理早于 30/60/90 天的 rollout JSONL（二次确认），显示 `codex --version`，检查 npm 上 `@openai/codex` 最新版，并复制升级命令。
- **MCP（只读）**：列出 `config.toml` 中的 `[mcp_servers.*]`，并可打开配置文件编辑。

> Sparkle 自动更新尚未接入（待确定托管方案），「检查更新」菜单项目前为置灰占位。

## 安装

### 下载发布版本

从 [Releases](https://github.com/qiaob/baobox_mac/releases) 页面下载最新的 `.zip`，解压后将 `Baobox.app` 拖入「应用程序」文件夹。

未公证的构建版本首次启动会被 Gatekeeper 拦截——右键点击 App 选择「打开」，或在终端执行：

```bash
xattr -cr /Applications/Baobox.app
```

### 从源码构建

Xcode 工程由 `project.yml` 通过 [XcodeGen](https://github.com/yonaskolb/XcodeGen) 生成，不纳入版本控制：

```bash
brew install xcodegen
xcodegen generate
open Baobox.xcodeproj
```

在 Xcode 中选择 `Baobox` scheme 运行，或直接使用项目自带的 Makefile：

```bash
make dev      # Debug 编译并直接运行，不影响 /Applications 中的版本
make install  # Release 编译并安装到 /Applications，随后自动启动
```

## 权限说明

首次启动会引导授权以下系统权限（系统设置 → 隐私与安全性）：

| 权限 | 用途 | 未授权时 |
|---|---|---|
| 屏幕录制 | 截取屏幕内容 | 截图工具不可用 |
| 辅助功能 | 模拟 ⌘V 自动回填粘贴剪贴板历史；移动 / 缩放窗口 | 剪贴板降级为仅复制；窗口管理不可用 |
| 麦克风 | 录屏时录制人声（可选，仅在开启该选项时才会请求） | 录屏继续进行，但不含麦克风音轨 |

所有权限均绑定 bundle id `com.baobox.app`；修改 bundle id 会导致系统要求重新授权。授权后立即生效，无需重启。
