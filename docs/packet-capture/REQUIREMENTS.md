# 网络抓包工具 — 需求文档

> 版本：v1.0（2026-07-23）· 设计者：Fable · 实现者：Opus 4.8
> 配套：同目录 `TECH_DESIGN.md`（技术方案）
> 定位：Baobox 的一个 `ToolModule`（`id: "netcapture"`，名称「网络抓包」，symbol `network`）。对标 Charles / Proxyman / mitmproxy 的核心能力，做成**菜单栏一键开关、UI 直观、能喂给 AI** 的本地抓包器。

## 1. 背景与定位

开发者调试网络请求（尤其手机 App）需要抓 HTTPS 包。现有工具（Charles/Proxyman）要么收费、要么常驻吃内存、要么配代理/装证书繁琐。用户诉求（原文提炼）：

1. **Mac 本地抓包** + **手机（Android/iOS）设代理后抓包**，都要**支持 HTTPS**。
2. **方便看到代理 IP**，方便手机填代理；**能一键 ADB 设置**更好。
3. **支持 HTTPS 证书安装**，一切以方便为主。
4. **有开关**开启/关闭，**不能一直开着浪费内存**。
5. UI 参考成熟抓包工具；**结合 AI**：给 AI 提供查询抓包结果的工具，或一键发送给 UI——**可以是一个单独的 MCP 开关，启动一个本地 MCP**。

实现取舍：以 macOS 原生 **Network.framework** 自建 HTTP(S) 中间人代理（MITM），证书签发借助系统自带 `/usr/bin/openssl`，**不打包第三方大体积运行时**（与 Baobox 一贯的原生、轻量、非沙盒定位一致）。用户已明确「开源项目或自研均可」，本设计选自研原生方案以便签名/公证与体积可控。

## 2. 术语

- **代理（Proxy）**：本工具在本机监听的 HTTP 代理端口，客户端（Mac 系统 / 手机）把流量指过来。
- **MITM**：对 HTTPS，用本地根证书（CA）动态签发目标域名的叶子证书，解密后再转发，从而看到明文。
- **Flow（流）**：一条被捕获的 HTTP 事务（请求 + 响应 + 元数据）。
- **CA**：本工具生成的本地根证书，装到设备并信任后才能解密该设备的 HTTPS。

## 3. 功能清单（按常用度排序）

### P0 — 核心抓包（里程碑一）

| # | 功能 | 说明 | 验收标准 |
|---|---|---|---|
| 1 | 代理开关 | 菜单/窗口一键「开始/停止抓包」。开 = 启动本地代理监听（默认端口 9090，可改）；关 = 停止监听、还原系统代理、释放缓冲。**关闭状态零后台开销**（不监听、不占内存缓冲） | 开→客户端流量被捕获；关→端口释放、系统代理还原、内存回落 |
| 2 | Mac 本地抓包 | 开启时自动把**当前活跃网络服务**的 HTTP/HTTPS 系统代理指向 `127.0.0.1:<port>`；关闭时还原为开启前的状态（含原本就有的代理设置） | 开启后 Safari/curl 等遵守系统代理的流量出现在列表；关闭后系统代理恢复原样 |
| 3 | HTTPS 解密（MITM） | 装并信任 CA 的设备，其 HTTPS 明文可见；未信任的域名/连接**放行不拦**（透传，不报错、不中断） | 已信任设备访问 https 站点，列表显示明文 URL/头/体；未装证书设备照常上网（透传） |
| 4 | Flow 列表与详情 | 窗口左列表（方法、状态码、host、path、耗时、大小、时间），右详情（概览 / 请求 / 响应 / 头 / Body，JSON 美化、图片预览）；实时追加 | 抓到的请求 1s 内进列表；点击看到完整请求/响应 |
| 5 | 搜索 / 过滤 / 清空 | 按 host/path/方法/状态码过滤，关键字搜索 URL 与 body；一键清空 | 过滤即时生效；清空后列表空、内存释放 |

### P0 — 手机抓包便利（里程碑一，与上并行）

| # | 功能 | 说明 | 验收标准 |
|---|---|---|---|
| 6 | 代理 IP 展示 | 窗口顶部醒目显示本机所有可用局域网 IPv4 + 端口（如 `192.168.1.23 : 9090`），一键复制；多网卡列全 | 手机与 Mac 同网，按展示的 IP:端口填代理即可连上 |
| 7 | 证书下载页 + 二维码 | 设备把代理设成本机后，浏览器访问magic 域名（`http://baobox.proxy/`）即可下载 CA 证书；窗口显示该 URL + **二维码**，手机扫码直达 | 手机扫码/访问该地址能下到 `.crt`/`.pem`；iOS 得到描述文件 |
| 8 | Mac 证书一键安装并信任 | 一键把 CA 装进登录钥匙串并设为受信任根（需管理员授权，走系统授权弹窗）；显示当前信任状态；一键移除 | 点击授权后 Mac 上 HTTPS 解密生效；移除后 CA 不再受信 |

### P1 — ADB 与增强（里程碑二）

| # | 功能 | 说明 | 验收标准 |
|---|---|---|---|
| 9 | Android 一键设置（ADB） | 检测到 `adb` 与已连设备：一键 `settings put global http_proxy <ip>:<port>` 设代理、推送 CA 到设备下载目录并拉起证书安装界面；一键清除代理（`:0`）。未装 adb / 无设备时给出安装指引 | 连 USB 调试的设备点一下即设好代理；再点清除代理还原 |
| 10 | 本地 MCP 开关 | **单独开关**启动一个本地 MCP 服务（HTTP，127.0.0.1:<mcpPort>），暴露查询抓包结果的工具供 AI 调用；一键把该 MCP 注册进 Claude Code / Codex 配置 | 开关开→AI（Claude Code/Codex）可通过 MCP 列出/检索/读取 flow；关→端口释放、配置可一键移除 |
| 11 | 发送给 AI / 复制 | 选中 flow 右键：复制为 cURL、复制为 Markdown（请求+响应摘要）、「发送到 Claude Code」（开终端把该 flow 作为提示词喂给 claude） | 复制内容可直接粘进终端/AI；发送能在终端带上下文启动 |
| 12 | 断点 / 改写（观察池，暂不做） | 拦截请求/响应做修改——价值高但交互复杂，本期只做只读抓包，留接口位 | —— |

## 4. 交互与 UI（菜单、窗口、样式、配色）

### 4.1 菜单栏二级菜单（常用度顺序）

```
[状态行,置灰] 抓包中 · 127.0.0.1:9090 · 42 条        （停止时：「未开启」）
──────
● 开始抓包 / ■ 停止抓包                （主开关，切换态；开时显示停止）
打开抓包窗口…                          → 中心窗口
──────
代理地址：192.168.1.23:9090            （置灰信息行，点击复制；多 IP 取首个 + 窗口看全部）
安装 HTTPS 证书                        → submenu：Mac 一键安装并信任 / 显示证书下载二维码 / 通过 ADB 推送
──────
本地 MCP：关 / 开（开关，右侧 switch）  （开时副标题显示 127.0.0.1:<mcpPort>）
──────
（框架自动追加「网络抓包设置…」）
```

- 未开启时状态行「未开启」，仅「开始抓包 / 打开窗口 / 证书 / MCP / 设置」可用；代理地址行在开启时才有意义（关闭也可显示本机 IP 供提前告知手机）。
- 主开关切换即时反映（`menuNeedsUpdate` 重建）。

### 4.2 中心窗口「抓包」（Proxyman 式两栏）

尺寸默认 980×620，titled/closable/resizable。自上而下：

- **顶部工具条**：`● 开始/停止`（主色按钮，运行时红点+“停止”）、`清空`、`搜索框`（占位「搜索 URL / body」）、方法过滤 chips（ALL/GET/POST/PUT/DELETE/其它）、状态码过滤（2xx/3xx/4xx/5xx）、右侧「代理 `IP:端口` + 复制 + 二维码按钮」、「证书」下拉、「MCP ●/○」小徽标。
- **左栏 Flow 列表**（宽约 420pt，可拖）：每行——方法徽标（配色见下）、状态码（配色）、`host`、`path`（截断）、右侧小字：耗时 `123ms`、大小 `4.2KB`、时刻 `HH:mm:ss`。滚动跟随最新（可锁定）。
- **右栏详情**（Tab）：
  - 概览：方法/URL、状态、协议、远端 IP、开始/结束/耗时、请求&响应大小、Content-Type。
  - 请求：请求行 + 头表 + Body（按 Content-Type：JSON 折叠树/美化、表单键值、文本、二进制标注大小与可另存）。
  - 响应：状态行 + 头表 + Body（JSON 美化、图片直接预览、HTML/文本、gzip 自动解压后展示）。
  - 原始：请求/响应原始报文（可复制）。
- **底部状态条**：总条数 / 已过滤条数、代理运行状态、MCP 状态、内存占用估算。
- **空状态**：未开启时中间大图 + 「点击开始抓包」+ 三步引导（开代理 → 设备设代理 → 装证书），旁附代理 IP 与二维码。

### 4.3 配色（复用 `docs/design/ui-design-v1.html` 令牌）

- 主强调色 accent：浅色 `#17A398` / 深色 `#2BC4B8`（按钮、选中、二维码前景、运行中指示）。
- 面板/分隔/次要文字沿用 `--surface / --line / --muted / --faint`，深浅两套自动适配（`@AppStorage` 主题或系统）。
- **HTTP 方法徽标配色**（浅底彩字/彩底白字，圆角 4）：GET 绿 `#28A745`、POST 蓝 `#0A84FF`、PUT 橙 `#F59E0B`、DELETE 红 `#FF3B30`、PATCH 紫 `#8B5CF6`、其它灰。
- **状态码配色**：2xx 绿、3xx 蓝、4xx 橙、5xx 红、无响应/错误 灰红。
- 运行状态点：运行中 `#28C840`（绿）呼吸，停止灰；MITM 未信任透传的 flow 用次要色标注「未解密」。

## 5. 非功能需求

- **性能与内存**：关闭时零常驻开销（不监听、无缓冲）。开启时 flow 列表内存有上限（默认 1000 条，可配 200–5000），超限淘汰最旧；单条 body 捕获上限（默认 5MB，超出透传但标「已截断」，不整包驻留）。目标：抓包开启常态额外内存 < 80MB（中等流量）。
- **隐私**：抓包内容默认**仅内存**，不落盘；用户可显式「导出 .har」。CA 私钥仅存本机 supportDir，权限 600。工具会解密 HTTPS，UI 明确提示「仅用于本机调试，请勿抓取他人隐私」。关闭或退出即清空内存 flow（可选保留至下次）。
- **稳定性**：单条连接解析失败只丢该 flow，不影响代理与其它连接；无法 MITM 的连接（未信任/证书固定 Pinning）**透传**而非中断，保证设备正常上网。App 退出/崩溃时务必还原系统代理（`willTerminate` + 兜底）。
- **兼容**：HTTP/1.1 明文与 TLS；ALPN 只协商 `http/1.1`（不做 HTTP/2 解析，强制 h2 的少数站点会透传或降级，UI 标注）。WebSocket、HTTP/2、QUIC 不在本期解析范围（透传）。
- **体验**：完整键盘操作、浅/深色适配、界面中英双语（`netcapture.*` 词条，遵循 `L()` 约定）。
- **权限**：设系统代理用 `networksetup`（用户级，无需管理员）；信任 CA 需一次管理员授权；抓包本身不需要「屏幕录制/辅助功能」等既有权限。绑定 bundle id `com.baobox.app`。

## 6. 里程碑

| 里程碑 | 内容 | 验收 |
|---|---|---|
| M1 | 代理开关 + Mac 本地抓包 + HTTPS MITM + Flow 列表/详情/搜索 + 代理 IP/二维码 + Mac 证书一键信任 | 能抓 Mac 与手机（手动设代理 + 扫码装证书）的 HTTPS 明文 |

## 7. 明确不做（本期）

- HTTP/2 / QUIC / WebSocket **解析**（透传，不解密展示）。
- 断点拦截与请求改写、请求重发/编辑、Map Local/Remote、限速。
- 抓包内容云同步、团队共享。
- 绕过证书固定（Certificate Pinning）——被 Pin 的连接透传，UI 说明这是预期行为。
