# 键盘点击（Keyboard Navigation）需求文档

> 对标 [Homerow](https://www.homerow.app)。作为 Baobox 的一个新工具模块接入（菜单栏 / 快捷键 / 设置统一入口）。

## 1. 背景与目标

用键盘完成鼠标操作，手不离键盘。核心是「用键盘点击屏幕上任意可交互 UI 元素」：按快捷键后，所有可点击元素叠加字母标签，输入字母即「点击」。

## 2. 术语

- **Hint / 标签**：叠加在可点击元素上的字母提示（如 `a`、`sd`）。
- **可点击元素**：按钮、链接、复选框、单选、弹出按钮、菜单项、标签页、输入框等可交互 AX 元素。
- **AX**：macOS Accessibility API（`AXUIElement`）。

## 3. 功能需求

### 3.1 Click Mode（点击模式）—— 核心 / MVP

- **触发**：全局快捷键（出厂**不绑定**，用户在快捷键页自设，避免与系统/其它 App 冲突）。
- **流程**：
  1. 枚举当前**前台 App** 的所有可点击元素及其屏幕位置。
  2. 每个元素叠加一个字母标签（透明 overlay）。
  3. 用户按标签字母（可多字母），实时**前缀过滤**、高亮候选、隐藏不匹配。
  4. 唯一匹配 → **点击**该元素 → 关闭标签层。
  5. `Esc` 或点击标签层外 → 取消。
- **点击方式**：优先 **AXPress**（可访问性点击，最稳、不移动真实鼠标）；元素不支持时**合成鼠标事件**（移动到元素中心 + 左键按下抬起）。

### 3.2 Scroll Mode（滚动模式）—— P1

- 触发：另一快捷键。vim 式键盘滚动：`j/k` 行、`d/u` 半屏、`gg/G` 顶/底；`Esc` 退出。对当前焦点滚动区发送滚动事件。

### 3.3 Search（搜索）—— P2

- 搜索屏幕可见文本/元素，回车跳转或点击。

## 4. 非功能需求

- **权限**：辅助功能（Accessibility）。未授权时菜单显示一条置灰引导（复用 `Permissions`），不报错。
- **性能**：AX 枚举在后台线程，目标 200ms 内出标签；大 App（浏览器/Xcode）有数量/深度/超时护栏，超限截断而非卡死。
- **不破坏原状态**：标签层不夺取真实焦点、不改变原 App 状态；点击后焦点回到目标处，overlay 立即消失。

## 5. 非目标（本期不做）

- **Web App 深度支持**：Chrome/Safari 网页内元素的 AX 暴露有限（需特殊开启），Homerow 为此做了大量专门工作。本期只覆盖**原生 macOS App** + 系统已暴露的基本 web AX。
- Mission Control 导航、标签主题、声音反馈、hyperkey 等周边特性。
- 被遮挡元素的可见性检测（MVP 不做遮挡剔除）。

## 6. 分阶段

| 阶段 | 内容 |
|---|---|
| **MVP** | Click Mode（原生 App） |
| **P1** | Scroll Mode、忽略应用名单、设置项 |
| **P2** | Search、web app 增强、标签主题、系统菜单栏/Dock 元素 |
