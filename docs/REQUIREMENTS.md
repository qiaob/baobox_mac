# Mac 工具集（暂定名 Baobox）需求文档

> 版本：v0.1（2026-07-20）
> 状态：已评审——技术栈、分发方式、系统版本、功能范围均已确认

## 1. 背景与定位

业界的 Mac 效率工具（截图、剪贴板管理等）都是单独的软件，每个都要单独下载、单独付费、单独常驻后台，比较麻烦。本项目做一个 **macOS 菜单栏常驻的工具集合 App**：一个 App 内置多个工具，统一入口、统一快捷键管理、统一设置。

- 对标产品：CleanShot X（截图）、Paste / Maccy（剪贴板）
- 长期形态：可持续新增工具模块的"工具箱"，功能内置（非插件市场模式）

## 2. 已确认的技术决策

| 决策项 | 结论 | 理由 |
|---|---|---|
| 技术栈 | Swift + SwiftUI 原生 | 截图/剪贴板深度依赖系统 API（ScreenCaptureKit、NSPasteboard、辅助功能）；原生常驻内存最小（~30MB vs Electron 150MB+）；只做 macOS，不需要跨端 |
| 分发方式 | 官网 / GitHub Releases 直接分发 | 不受 App Store 沙盒限制（辅助功能模拟粘贴、部分截图能力沙盒内做不了）；用 Sparkle 做自动更新；需 Apple 开发者账号做签名 + 公证 |
| 最低系统版本 | macOS 14 (Sonoma)+ | 可用完整的 ScreenCaptureKit 能力和 SwiftUI 新特性，无需兼容分支 |
| 功能目标 | 对标成熟软件（CleanShot X + Paste 主要能力） | 按里程碑分阶段交付，每个里程碑都是可用状态 |

## 3. 整体架构要求

工具集合的核心是**框架先行**，后续加工具是加模块而不是改架构：

- **App 形态**：菜单栏常驻（`MenuBarExtra` / NSStatusItem），无 Dock 图标（可在设置中开关），开机自启动可选
- **菜单栏菜单 = 纯工具列表**：每个工具一行（图标 + 名称 + 主快捷键），悬停展开二级菜单承载该工具的操作/历史/设置入口；主行单击执行该工具的默认动作。菜单中不内嵌任何工具的具体内容（如剪贴板预览），保证工具数量增长后版式不变；通用项（设置、检查更新、退出）用分隔线沉底
- **工具模块化**：每个工具实现统一的 `ToolModule` 协议（名称、图标、设置页、快捷键声明、启用开关），向框架注册
- **全局快捷键中心**：统一注册/冲突检测/自定义，持久化到设置
- **统一设置面板**：通用设置（启动、更新、外观）+ 每个工具一个设置 Tab
- **权限引导**：首次使用引导用户授予"屏幕录制"和"辅助功能"权限，检测权限状态并给出跳转系统设置的入口
- **更新**：集成 Sparkle，支持自动检查更新

## 4. 工具一：截图

### P0（核心，里程碑 M1）
- **智能截图，单一全局快捷键**（默认 ⌘⇧2），唤起后自动检测意图，无需事先选模式：
  - 悬停：实时命中检测窗口（CGWindowList）并整体高亮，标注 App 名与窗口尺寸；**单击 = 截取该窗口**
  - **按下拖拽（超过约 4px 阈值）= 区域截图**：八向手柄微调、方向键逐像素移动（⇧ ×10）、实时尺寸标注、像素放大镜
  - **⏎ = 当前屏幕全屏截图**；Esc 取消
  - Overlay 底部常驻提示条，列出当前状态可用按键
- 多显示器支持（每屏分别截取）
- 截图结果：复制到剪贴板 / 保存到指定目录（文件名模板可配置）
- 技术：ScreenCaptureKit；需要"屏幕录制"权限

### P1（里程碑 M2）
- ✅ 截图后标注编辑器（已实现，选区原地标注，Snipaste 式工具条）：矩形、椭圆、箭头、画笔、荧光笔、马赛克、文字、橡皮（整笔删除）、撤销重做（⌘Z/⇧⌘Z）、三档粗细 + 七色色板；选定工具即冻结选区画面（SCK 排除自身窗口），导出为冻结帧 + 标注合成
- ✅ 贴图（Pin）：把截图（含标注）钉在屏幕最上层，可拖动、双击关闭；滚轮等比缩放（0.2×–5×，锚定鼠标位置）、⌥+滚轮调透明度；右键复制/另存/恢复原始大小/透明度档位；菜单提供「从剪贴板贴图」「关闭全部贴图」
- ✅ 像素放大镜（P0 遗留补齐）：悬停/框选/拉手柄时跟随光标，17×17 物理像素 ×8 放大、十字线与中心像素格，附全局坐标与颜色 Hex 读数（底图为会话开始时冻结的整屏画面）
- ✅ 截图历史（已实现）：每次截图（复制/保存/贴图）自动存档，保留张数可设（默认 20），菜单缩略图列表支持复制/贴到屏幕/另存/删除/清空

### P2（里程碑 M3）
- 滚动截图（长网页/长文档）
- ✅ 屏幕录制（已提前实现）：复用截图选区交互（框选 / 点选窗口 / ⏎ 全屏，后两者也先转为选区待确认），SCStream → H.264 MP4（30fps）。选区右下角**确认工具条**（系统声音、麦克风开关 + 取消 + 开始录制，均可鼠标点击）；录制中红色边框圈出范围（描边在选区外沿），右下角控制条含红点计时 + **暂停/继续**（时间轴平移补偿，成片无空洞）+ 停止 + 取消不保存，可拖动、放不下自动换位，边框与控制条均从画面中剔除。**声音**：系统声音走 SCK（无需额外权限）；麦克风走 AVCaptureSession 独立第二音轨（需系统授权 + hardened runtime 的 audio-input entitlement），拒绝授权时降级继续录。完成后 Finder 定位文件。**输出**：MP4 或 GIF（≤10fps、宽度 720px、无声、适合短片段，长片段自动降帧率控制在 300 帧内）；系统声音+麦克风同录时默认离线混为单条音轨（视频直通不转码），兼容只播首轨的播放器
- OCR 取字（Vision framework，识别后复制文本）
- 自拷贝到云/生成分享链接（待定，需要后端，暂不承诺）

## 5. 工具二：剪贴板管理

### P0（核心，里程碑 M1）
- 后台监听剪贴板（NSPasteboard 轮询），记录历史：纯文本、富文本、图片、文件路径、链接
- 全局快捷键唤出历史面板（浮层，键盘上下选择 + 回车粘贴）
- 搜索过滤（按内容/类型）
- 选中后自动回填粘贴到当前 App（辅助功能权限 + CGEvent 模拟 Cmd+V；无权限时降级为仅复制）
- 历史条数上限可配置，超限淘汰

### P1（里程碑 M2）
- ✅ 置顶 / 收藏常用条目（⌘P 置顶，排序靠前、不被上限淘汰与过期清理）
- ✅ 隐私保护：忽略 `org.nspasteboard.ConcealedType` / `TransientType` 标记内容；按 App 忽略名单（设置页添加/移除，以变更时前台 App 判定来源）
- ✅ 过期自动清理（永久/1/7/30/90 天可选，启动、新增、开面板时触发）
- ✅ 纯文本粘贴模式（面板内 ⌥⏎ 单次去格式；全局 ⌘⌥V 纯文本粘贴最近一条）
- ✅ 单条删除（⌘⌫ 或行内悬停删除按钮，配合隐私场景精确清除）

### P2（里程碑 M3）
- 片段（Snippets）：预置常用文本，支持关键词展开
- iCloud 多设备同步
- 规则化处理（如复制链接自动去跟踪参数，待定）

## 6. 后续工具规划（已评审排期）

每个新工具只需实现 `ToolModule` 协议接入，不改动框架。按"复用现有基础设施程度 + 实现成本"排定优先级：

### 已排入 M2（与标注/贴图/收藏并行）
- ✅ **取色器**：屏幕任意位置取色（NSColorSampler，无需额外权限），取色后按设置格式（Hex/RGB/SwiftUI）自动复制，历史色板持久化，菜单二级入口展示最近颜色。出厂不绑定快捷键（默认曾用的 ⌘⇧C 与浏览器开发者工具"检查元素"冲突），用户在快捷键页自行设置
- ✅ **防休眠**：一键阻止系统休眠（IOPMAssertion），支持 15 分钟/1 小时/2 小时/无限期定时，可选同时防显示器休眠
- ✅ **窗口管理**：快捷键贴半屏/四分屏/最大化/居中（出厂不绑定键位——常用的 ⌃⌥ 系列与 Rectangle 完全同键，用户在快捷键页自行设置；菜单入口始终可用），复用已申请的辅助功能权限；**必须完整兼容多显示器**——布局以窗口当前所在屏幕的可用区域为基准，支持把窗口在显示器之间循环移动（等比缩放适配目标屏），支持恢复窗口原始位置

### M2 后追加实现
- ✅ **二维码生成**（独立模块）：⌃⇧Q 唤起浮层，自动带入剪贴板文字、可编辑实时重绘（纠错 M、含静区），支持复制图片/保存 PNG/钉在屏幕上；纯本地 CIQRCodeGenerator，无需权限
- ✅ **窗口布局快照**（并入窗口管理）：菜单「保存当前布局…」记录所有常规 App 非最小化窗口的位置尺寸，菜单点击一键恢复（标题优先匹配、顺序兜底；未运行的 App 跳过），设置页可删除。**多显示器**：每条记录存所在显示器 UUID + 屏内相对位置，恢复时原屏还在则按相对位置精确还原（分辨率/排列变化不影响），原屏拔掉则夹回现有屏幕可见区域
- ✅ **Cursor / Codex 助手**（详见 `docs/cursor-codex-assistant/`）：把本地 Codex CLI 与 Cursor 编辑器的状态与配置收进菜单栏——最近 Codex 会话浏览与一键终端续接（`codex resume`）+ 轻量会话窗口（搜索/续接/复制命令/删除）；Codex 配置可视化（approval_policy / sandbox_mode / model 单选，对 config.toml 做保注释的行级读写，不可编辑值自动置灰）；可选把 Baobox 通知程序写入 config.toml `notify`，回合结束发系统通知；Cursor Rules 项目管理（枚举 `.cursor/rules/*.mdc`、旧式 `.cursorrules` 检测、内置通用/前端/Python 模板一键写入）；Cursor 全局 MCP（`~/.cursor/mcp.json`）面板增删。纯本地文件解析，不调用任何 AI API、无需登录，写用户文件保留未知内容并备份 `.baobox.bak`
- ✅ **Claude Code 助手**（详见 `docs/claude-code-assistant/`）：把本地 Claude Code CLI 的状态与配置收进菜单栏——会话状态/续接、5 小时额度窗口与今日估算花费；中心窗口含按天/项目/模型用量报表、调用统计（Skill/斜杠命令、MCP 两级、内置工具）与当日改动审计；可选安装事件通知与危险命令卫士 hooks；配置节以可视化控件编辑 settings.json（权限模式/模型/保留天数/权限预设/隐私开关/Co-Authored-By/CLAUDE.md）、statusline 生成器、MCP 面板、磁盘清理与版本检查。纯本地文件解析，不调用任何 AI API、无需登录，写用户 JSON 保留未知字段并备份 `.baobox.bak`

### M3 候选
- **屏幕 OCR 取字 / 二维码识别**：复用截图选区 Overlay 与权限，Vision 框架识别（原截图 P2 提为独立工具入口）
- **文本片段（Snippets）**：与剪贴板管理协同（原剪贴板 P2）

### 观察池（价值待验证，暂不排期）
- 开发者工具箱（JSON 格式化/时间戳/Base64/UUID）、单位换算、菜单栏日历

### 明确不做
- 系统清理、CPU/网速监控、笔记类——与"效率工具集"定位不符或需另立产品

## 7. 非功能需求

- **性能**：常驻内存 < 50MB；剪贴板监听 CPU 占用无感知；截图唤起 < 200ms
- **隐私**：所有数据默认仅存本地（剪贴板历史落盘需加密或明确告知）；不采集用户内容
- **稳定性**：单个工具异常不拖垮整个 App；剪贴板历史持久化，重启不丢失
- **体验**：完整键盘操作支持；浅色/深色模式适配；界面语言先中文，预留本地化

## 8. 里程碑规划

| 里程碑 | 内容 | 验收标准 |
|---|---|---|
| M1 | 框架（菜单栏、快捷键中心、设置、权限引导）+ 截图 P0 + 剪贴板 P0 | 日常可用：能截图保存/复制，能唤出剪贴板历史并回填粘贴 |
| M2 | 截图标注/贴图/历史 + 剪贴板收藏/隐私过滤/清理 + Sparkle 更新 | 覆盖 CleanShot/Paste 的高频日常场景，可对外分发 |
| M3 | 滚动截图、录屏、OCR、Snippets、iCloud 同步 | 主要能力对齐成熟软件 |
